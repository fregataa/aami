# Architecture Decision Records (ADR)

This directory contains Architecture Decision Records (ADRs) that document key architectural decisions for the AAMI project.

## What is an ADR?

An ADR is a document that records significant decisions affecting software architecture. Each ADR includes:

- **Context**: Background and reasons for needing a decision
- **Decision**: The chosen decision and rationale
- **Consequences**: Advantages and disadvantages of the decision

## Status Definitions

- **Proposed**: Under review
- **Accepted**: Approved and being implemented
- **Deprecated**: No longer recommended
- **Superseded**: Replaced by another ADR
